${symbol_pound}${symbol_pound} 十五、实现要求

${symbol_pound}${symbol_pound}${symbol_pound} client 层

- 定义 Command / Query / DTO / Result。
- 如需要 gRPC，维护 proto 契约。
- 不允许引入 Mapper、Repository、DO、Redis、数据库访问等实现细节。

${symbol_pound}${symbol_pound}${symbol_pound} domain 层

- 定义领域对象、领域枚举、领域服务。
- 表达核心业务规则。
- 不允许引入 Spring Web、MyBatis、Redis、gRPC、Sa-Token。

${symbol_pound}${symbol_pound}${symbol_pound} infrastructure 层

- 定义 DO / Mapper / Repository / Gateway / Converter。
- 对象转换优先使用 MapStruct。
- 不允许编排完整业务流程。
- 不允许反向调用 adapter。

${symbol_pound}${symbol_pound}${symbol_pound} app 层

- 编排用例流程。
- 控制事务边界。
- 调用领域对象和基础设施能力。
- 不允许写 HTTP Controller、gRPC Service、MQ Listener。
- 不允许向 adapter 泄漏 DO。

${symbol_pound}${symbol_pound}${symbol_pound} adapter 层

- 提供 HTTP / gRPC / MQ 接入。
- 只做协议转换、参数校验、调用 app。
- 不允许直接访问 Mapper。
- 不允许写完整业务编排。

${symbol_pound}${symbol_pound}${symbol_pound} start 层

- 只做启动装配和配置。
- 不允许写业务逻辑。
